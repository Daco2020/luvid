# 핵심 기능 1: 나 사용 설명서 (Level 1) - 상세 기획서

> "연애가 서툴러도 괜찮아요. 천천히, 나를 알아가는 것부터 시작해요."

## 1. 기획 의도

### 1.1 왜 이 기능이 필요한가?

우리의 타깃 유저들(민지, 준호, 수빈, 지훈, 예린)은 공통적으로 **"나는 어떤 사람일까?"**라는 질문에 답하지 못합니다.
기존 심리 테스트들은 **MBTI, 애착유형** 같은 라벨만 붙여줄 뿐, **"그래서 나는 연애에서 어떻게 행동해야 하는가?"**에 대한 답은 주지 못합니다.

Luvid는 관계 심리학(Window of Tolerance, Polyvagal Theory)에 기반하여 사용자 자신의 **'작동 방식(Operating Instructions)'**을 체계화합니다.

### 1.2 Level 1의 목표

**완료 후 사용자가 얻는 것**:

1. **자기 이해**: "나는 스트레스 받으면 동굴로 들어가는구나" (신경계 반응 이해)
2. **소통 도구**: "그럴 땐 나를 그냥 둬줘"라고 말할 수 있는 근거 확보
3. **관계 효능감**: 내 감정을 내가 조절할 수 있다는 자신감

**정체성 호칭 부여**:

- 완료 시: **"준비된 연인(Ready-to-Date)"** 뱃지 획득

## 3. 섹션 1: 감정 패턴 발견하기 (Maintenance Manual)

### 3.1 목표

사용자의 **'윈도우 오브 톨러런스(Window of Tolerance)'**를 파악하여, 최적의 상태를 유지하고 스트레스 상황에서 회복하는 **'유지 보수 매뉴얼'**을 완성하는 것.

### 3.2 심리학적 배경

- **윈도우 오브 톨러런스 (Dan Siegel)**: 각성의 최적 영역. 이 범위를 벗어나면 과각성(분노/불안) 혹은 저각성(우울/무기력) 상태가 됨.
- **폴리베이갈 이론 (Stephen Porges)**: 신경계의 3단계 반응 (사회적 참여 -> 가동화 -> 부동화).

### 3.3 질문 구성 (총 10문항)

**[Part 1: 기준점 및 에너지 (Operating Environment)]**

#### Q1. 나의 연료 (Energy Source)

```
질문: 당신이 가장 '나다운' 상태로 충전되는 순간은 언제인가요?

A) 혼자 조용한 공간에서 쉴 때 (Solitude)
   → 내향형 충전
B) 친한 친구 1-2명과 깊은 대화를 할 때 (Intimacy)
   → 관계형 충전
C) 밖에서 사람들과 어울리고 활동할 때 (Social)
   → 외향형 충전
D) 좋아하는 취미나 몰입할 거리가 있을 때 (Activity)
   → 활동형 충전
```

#### Q2. 최적의 시간 (Optimal Time)

```
질문: 중요한 대화를 하기에 가장 좋은, 당신의 '골든 타임'은?

A) 머리가 맑은 '오전 시간'
B) 일과를 마친 '저녁 시간'
C) 배부르고 여유로운 '식사 후'
D) 주말이나 휴일 같은 '쉬는 날'
```

**[Part 2: 경고 신호 및 트리거 (Triggers & Warning Signs)]**

#### Q3. 외부 트리거 (External Triggers)

```
질문: 데이트 중, 당신의 기분을 순식간에 망치는 것은? (가장 예민한 부분)

A) 시끄러운 소음이나 복잡한 인파 (감각 과부하)
B) 배고픔이나 더위/추위 (신체적 불편함)
C) 계획이 틀어지거나 갑자기 변동될 때 (통제력 상실)
D) 상대방이 핸드폰만 볼 때 (무시당하는 느낌)
```

#### Q4. 나의 경고등 (Warning Signs)

```
질문: 스트레스를 받기 시작하면, 겉으로 드러나는 당신의 첫 반응은?

A) 말이 없어지고 표정이 굳는다 (Shutdown)
   → 저각성 초기 신호
B) 목소리가 커지거나 말이 빨라진다 (Mobilization)
   → 과각성 초기 신호
C) 한숨을 쉬거나 안절부절못한다 (Anxiety)
   → 불안 반응
D) 냉소적이거나 비꼬는 말투가 나온다 (Defense)
   → 방어 기제
```

---

**[Part 3: 반응 패턴 (Fight, Flight, Freeze)]**

#### Q5. 불확실성 반응 (Uncertainty)

```
상황: 어제 데이트 후 "즐거웠어"라고 보냈는데, 하루가 지나도 답장이 없습니다. (읽음 표시 없음)

A) "바쁘겠지" 하고 내 할 일 한다. (Secure)
   → 안정적 처리
B) "내가 뭐 실수했나?" 계속 대화창을 확인한다. (Anxious)
   → 불안 반응 (과각성)
C) "답장 안 하면 나도 안 해" 하고 번호를 지운다. (Avoidant)
   → 회피/차단 반응
D) "왜 답장이 없어?"라고 문자나 전화를 바로 한다. (Protest)
   → 저항 반응
```

#### Q6. 과각성 반응 (Hyper-arousal)

```
상황: 연인과 말다툼 중, 감정이 주체할 수 없이 격해졌습니다. 이때 당신의 충동은?

A) 상대방을 이기기 위해 논리적으로 따진다. (Fight)
B) 화를 참지 못하고 소리를 지르거나 물건을 던지고 싶다. (Explosion)
C) "그만해!"라고 외치고 자리를 박차고 나간다. (Flight)
D) 눈물이 터져서 말을 잇지 못한다. (Flood)
```

#### Q7. 저각성 반응 (Hypo-arousal)

```
상황: 너무 지치고 힘들어서 에너지가 바닥났을 때(방전), 당신의 모습은?

A) 아무것도 안 하고 하루 종일 누워만 있는다. (Freeze)
B) 상대방의 말(위로)조차 듣기 싫고 귀찮다. (Withdrawal)
C) 멍하니 있거나 현실감이 없어진다. (Dissociation)
D) 무조건적인 지지와 응원만 받고 싶다. (Regression)
```

**[Part 4: 회복 매뉴얼 (Regulation & Recovery)]**

#### Q8. 자기 조절 (Self-Regulation)

```
질문: 격해진 감정을 스스로 진정시키는 가장 효과적인 방법은?

A) 혼자만의 시간을 가지며 동굴로 들어간다. (Space)
B) 심호흡을 하거나 찬물로 세수한다. (Physiological)
C) 맛있는 걸 먹거나 좋아하는 영상을 본다. (Sensory)
D) 감정을 글로 쓰거나 친구에게 털어놓는다. (Expression)
```

#### Q9. 상호 조절 (Co-Regulation)

```
질문: 당신이 힘들 때, 연인이 해주길 바라는 '최고의 위로'는?

A) "무슨 일이야?" 하고 끝까지 들어주기 (Listening)
B) 아무 말 없이 꽉 안아주기 (Physical Touch)
C) "내가 해결해줄게" 하고 문제 처리해주기 (Act of Service)
D) "잠시 혼자 쉴래?" 하고 배려해주기 (Respecting Space)
```

#### Q10. 접근 금지 (Do Not Disturb)

```
질문: 기분이 안 좋을 때, 이것만은 절대 하지 말아야 할 것은?

A) "왜 저기아?" 하고 계속 이유 캐묻기 (Intrusion)
B) "별거 아니네" 하고 내 감정 축소하기 (Dismissal)
C) "웃어봐" 하고 분위기 띄우려 하기 (Invalidation)
D) 논리적으로 따지거나 조언하기 (Lecturing)
```

## 4. 섹션 2: 갈등 스타일 (Conflict Style) - 인터랙티브 스토리

### 4.1 목표

사용자의 **TKI 갈등 유형**과 **사과 수용 언어**를 파악하여, 싸움의 규칙을 정립.

### 4.2 UI 컨셉: 분기형 인터랙티브 스토리

**차별화 포인트**:

- 일반적인 사지선다 대신 **몰입형 스토리텔링**
- 선택에 따라 상황이 분기되어 실제 갈등의 연쇄 반응 체험
- 게임처럼 재미있으면서도 심리학적 깊이 유지

### 4.3 스토리 선택 단계 (Step 0)

사용자는 먼저 4가지 시나리오 중 하나를 선택합니다.

```
"어떤 상황에서 당신의 갈등 스타일을 확인하고 싶으신가요?"

[4개의 카드 형태로 제시]

📱 시나리오 A: "갑자기 연락이 끊긴 연인"
   (20대 초반 감성 - 불확실성, 불안 애착)

💼 시나리오 B: "중요한 약속을 잊어버린 연인"
   (20대 후반 감성 - 책임감, 존중)

🏠 시나리오 C: "명절 가족 모임 vs 개인 시간"
   (30대 초반 감성 - 가족/개인 균형)

💰 시나리오 D: "돈 문제로 의견이 엇갈린 상황"
   (30대 후반 감성 - 재정, 미래 계획)
```

**주의**: 나이대는 명시하지 않음. 사용자가 공감되는 상황을 자유롭게 선택.

### 4.4 분기 구조 (각 시나리오당 6개 선택지)

선택한 시나리오에 따라 6단계 분기 스토리가 진행됩니다.

#### 예시: 시나리오 A - "갑자기 연락이 끊긴 연인"

**[분기 1: 초기 반응]**

```
상황: 어제 데이트 후 "즐거웠어"라고 보냈는데, 하루가 지나도 답장이 없어요. (읽음 표시 없음)

선택지:
A) "바쁘겠지" 하고 내 할 일을 한다
   → 안정형 (Secure)
B) "내가 뭐 실수했나?" 계속 대화창을 확인한다
   → 불안형 (Anxious)
C) "답장 안 하면 나도 안 해" 하고 번호를 지운다
   → 회피형 (Avoidant)
D) "왜 답장이 없어?"라고 문자나 전화를 바로 한다
   → 저항형 (Protest)
```

**[분기 2: A 선택 시]**

```
상황: 이틀째 되는 날, 연인에게서 "미안, 바빴어"라는 짧은 답장이 왔어요.

선택지:
A) "괜찮아, 무슨 일 있었어?" 하고 대화를 이어간다
   → 협력형 (Collaborating)
B) "그래, 알았어" 하고 간단히 답한다
   → 수용형 (Accommodating)
C) "이틀이나 걸렸네. 다음엔 미리 말해줘"
   → 경쟁형 (Competing)
```

**[분기 3-6: 계속 분기]**

- 사과 언어 선호도 측정
- 갈등 해결 방식 측정
- 감정 조절 패턴 측정

### 4.5 측정 항목

각 선택지는 다음을 측정합니다:

1. **TKI 갈등 스타일** (5가지)

   - Competing (경쟁형)
   - Avoiding (회피형)
   - Accommodating (수용형)
   - Collaborating (협력형)
   - Compromising (타협형)

2. **사과 언어** (5가지)

   - Expressing Regret (후회 표현)
   - Accepting Responsibility (책임 인정)
   - Making Restitution (보상)
   - Genuinely Repenting (진심 어린 뉘우침)
   - Requesting Forgiveness (용서 구하기)

3. **고트맨의 4가지 파멸의 기사**
   - Criticism (비난)
   - Defensiveness (방어)
   - Contempt (경멸)
   - Stonewalling (담쌓기)

### 4.6 구현 데이터 구조

```typescript
interface Scenario {
  id: string;
  title: string;
  description: string;
  ageGroup: "early20s" | "late20s" | "early30s" | "late30s"; // 내부용
  branches: Branch[];
}

interface Branch {
  id: number;
  situation: string;
  choices: Choice[];
}

interface Choice {
  id: string;
  text: string;
  patterns: {
    conflictStyle?: ConflictStyle;
    apologyLanguage?: ApologyLanguage;
    horseman?: Horseman;
  };
  nextBranchId?: number; // null이면 종료
}
```

## 5. 섹션 3: 핵심 가치관 (Values) - 이상형 월드컵

### 5.1 목표

관계의 나침반이 될 **Top 3 가치**와 **딜브레이커(Dealbreaker)** 식별.

### 5.2 UI 컨셉: 3단계 토너먼트

**차별화 포인트**:

- 추상적인 가치관을 **구체적인 행동**으로 변환
- 이상형 월드컵 방식으로 재미있게 우선순위 결정
- 긍정 가치 + 딜브레이커를 모두 측정

### 5.3 단계 1: 가치 칩 선택 (Value Chips Selection)

사용자는 30-50개의 가치 키워드 중 **8개**를 선택합니다.

```
"당신의 삶에서 중요한 가치를 8개 선택해주세요"

[칩 형태로 나열 - 탭하면 선택]

🏷️ 성장  🏷️ 안정  🏷️ 자유  🏷️ 신뢰
🏷️ 유머  🏷️ 인정  🏷️ 건강  🏷️ 사랑
🏷️ 부(Wealth)  🏷️ 공헌  🏷️ 모험  🏷️ 정직
🏷️ 가족  🏷️ 종교  🏷️ 소통  🏷️ 책임
...

[선택한 칩은 하단 바구니로 이동]
최소 8개 / 최대 8개
```

**가치 목록 예시 (총 40개)**:

### 5.4 단계 2: 긍정 가치 토너먼트 (Positive Values Tournament)

선택한 8개 가치 각각에 대해 **구체적인 행동 4개**를 미리 준비합니다.

#### 예시: "신뢰" 가치의 구체적 행동 4개

```
1. "약속 시간 5분 전에 도착하는 연인"
2. "핸드폰 비밀번호를 공유하자고 먼저 제안하는 연인"
3. "내 친구들과도 스스럼없이 친해지려는 연인"
4. "자기 실수를 솔직하게 인정하고 사과하는 연인"
```

**토너먼트 구조**:

- 8개 가치 × 4개 행동 = **32강 토너먼트**
- 랜덤 대진표 생성
- 사용자는 두 행동 중 더 중요한 것을 선택
- 16강 → 8강 → 4강 → 결승 진행

```
[32강 1경기]
A: "약속 시간 5분 전에 도착하는 연인" (신뢰)
vs
B: "힘들 때 말없이 안아주는 연인" (사랑)

[사용자 선택] → A 승

[32강 2경기]
C: "내 성장을 응원하고 지원하는 연인" (성장)
vs
D: "경제적으로 안정적인 연인" (안정)

...
```

**결과**:

- 우승: 1위 가치
- 준우승: 2위 가치
- 4강: 3-4위 가치
- 8강: 5-8위 가치
- 전체 순위 확인 가능

### 5.5 단계 3: 딜브레이커 토너먼트 (Dealbreaker Tournament)

이번에는 각 가치에 **반하는 행동 4개**를 준비합니다.

#### 예시: "신뢰" 가치에 반하는 행동 4개

```
1. "약속 시간에 자주 늦고 미안해하지 않는 연인"
2. "핸드폰을 절대 보여주지 않고 숨기는 연인"
3. "내 친구들을 만나기 싫어하는 연인"
4. "실수해도 변명만 하고 사과하지 않는 연인"
```

**토너먼트 구조**:

- 8개 가치 × 4개 반대 행동 = **32강 토너먼트**
- 랜덤 대진표 생성
- 사용자는 두 행동 중 **더 용납할 수 없는 것**을 선택
- 16강 → 8강 → 4강 → 결승 진행

```
[32강 1경기]
A: "약속 시간에 자주 늦고 미안해하지 않는 연인"
vs
B: "내 감정을 무시하고 자기 할 말만 하는 연인"

[사용자 선택] → B 승 (더 치명적)

...
```

**결과**:

- 우승: 가장 치명적인 딜브레이커 (1위)
- 준우승: 2위 딜브레이커
- 4강: 3-4위 딜브레이커
